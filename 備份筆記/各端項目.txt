控端

 一. 總覽 - 顯示收益總覽、賽事狀況、賽果異動、賽程中心資訊異動

   1. 收益總覽 - "結算進度"顯示有下過注的賽事的比分結算進度(沒有下過注的賽事的比分結算不會影響結算進度)，"結算損益"為已結算的注單之損益

   2. 賽事狀況 - 顯示全部賽事的 已開盤/未開盤的賽事數、已結算賽事數、有比分但未結算賽事數（含無下注的賽事），滾球中的賽事則顯示　已開盤/未開盤的滾球賽事數　及　滾球注單數　

　 3. 賽果異動 - 顯示來自賽程中心裡 "賽果中心" 提供的資料比分的變動結果

   4. 賽程中心資訊異動 - 顯示賽程中心裡有發生異動的項目的資訊含 異動時間、球種、異動項目、異動的詳細內容

 二. 賽事下載

 三. 賽事開盤 - 根據自賽事下載列表裡的資料設定各球種賽事的開盤(開放下注、開放玩法賠率)，並可查看 賽事風險預測(賽事的注單損益預估) 及 賽事所使用的參數名片資訊

 四. 風險監控 

   1. 即時注單 - 列出所有符合篩選資料的注單及資料(下注時間、下注者帳號、玩法、金額、注單輸贏狀態等)

   2. 注單檢索 - 顯示符合檢索資訊的某比注單資料

   3. 賽事報表 - 檢索特定範圍日期及球種列出有下注過的賽事的損益報表(含滾球注單)

   4. 預測 - 顯示貨量(玩法的總下注額) 以及根據賽事的所有注單預測出當 比賽的結果為甚麼狀態會產生多少損益的結果，

             例如: 1張注單下雙數 $10 賠率派彩 12，另1張下單數 $8 賠率派彩 10，風險預測會去根據所有單雙注單做計算，當賽果比分為雙數時損益為 $-4 、單數時損益為 $0

 五. 比分輸入 - 對已結束的比賽進行結算，結算完成的賽事注單才能計算損益及派彩結果 (同時更新 賽事報表)

 六. 預設參數 

   1. 參數名片 - 管理參數名片，每個名片對應一組設定 (例如: 單注最高金額、累積關盤金額、過關單開放等) ，每個 聯盟 對應一張名片(於 預設參數/聯盟套用 設定)

   2. 聯盟套用 - 設定聯盟要套用的名片，分 早餐(全/半場) 和 今日/滾球(全/半場)

   3. 過關參數設定 - 設定過關單單筆最高注額、累積關盤金額、最大過關串數

 七. 管理

   1. 公告設定 - 管理公告，可設定公告訊息、訊息範本管理、公告開放、跑馬燈效果等，開放的公告會顯示在客端

   2. 系統預設 - 可設定自動收單時間、單注/過關單的最低金額及最高派彩、聯盟過關串數及是否開放比分結算

   3. 取消注單 - 依 注單號 或 GID 搜尋特定注單後再操作是否註銷此注單

   4. 賽程紀錄 - 查詢賽事在特定期間的操作訊息(例如: 賽事下載 或 開盤行為是由誰操作的等資訊)

   5. 會員帳號 - 可以查看所有的會員帳號、後置碼、會員標籤訊息(來自 管理/會員標籤)

   6. 會員標籤 - 管理可給會員設定的標籤訊息 (例如: 此會員為有風險的會員)

賽程中心

 一. 賽程管理 

   1. 賽事對應 - 管理賽事的資料對應，如聯盟、隊伍、選手、賽事類別、直播對應以及 是否開放給控端下載以進行開盤 。

 二. XBB 資料庫管理

   1. 球種管理 - 根據 球種 管理基本玩法的開放 ，每個球種對應1個 SID

   2. 聯盟管理 - 根據 球種 管理從賽事對應加入的 聯盟資料 ， 每個 聯盟 對應1個 LID

   3. 隊伍管理 - 根據 球種 管理從賽事對應加入的 隊伍資料 ， 每個 隊伍 對應1個 TID

   4. 選手管理 - 根據 球種 管理從賽事對應加入的 選手資料 ， 每個 選手 對應1個 PID

   5. 類別管理 - 管理從賽事對應加入的 賽事類別 ， 每個 賽事類別 對應1個 Tag-ID

   6. 子玩法管理 - 管理基本玩法外的其他子玩法 ， 每個 子玩法 對應1個 子玩法-ID

 三. 訊源中心 

   1. Matrix - 顯示 賽況資訊(時間、賽事名稱、隊伍/選手名稱、賽況顯示方式)，

               賽況顯示方式有兩種 Matrix 和 Radar ，每個 賽事賽況 對應1個  Matrix ID

 四. 實況直播中心 

   1. GLIVE - 顯示 GLIVE 的直播影片資訊 (如 球種、隊伍、開始/結束時間以及是否正在直播)，

      並根據 GID (賽事 ID) 設定對應直播 ，對應到正確的 ID 後直播才會在客端顯示

 五. 賠率中心

   1. 沙巴 - 顯示 沙巴 所提供的玩法賠率資訊 (包含 單式、總入球、半全場、波瞻等) 提供的玩法賠率較多。

   2. Matrix - 顯示 Matrix 所提供的賠率資訊，但只有單式的賠率 (讓球、大小、標準盤、單雙)，再細分 滾球與非滾球，相較於沙巴提供的資訊較少


 六. 賽果中心 - 顯示 Matrix 提供的賽況結果，包含是否已完賽，上下半場比分，是否加時延場等

關於訊源

訊源有兩種 matrix 和 statescore
足球和網球同時有 matrix 和 statescore ，其他球種只有 matrix

且 statescroe 的資訊優先取用
但如果 statescroe 沒有提供 ，就會使用 matrix
